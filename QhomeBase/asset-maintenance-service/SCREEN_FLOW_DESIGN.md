## Business Rules

### Assignment Rules:
1. **Maximum Task Limit**: Each technician can have maximum **8 pending tasks** (ASSIGNED + IN_PROGRESS status)
   - System will **block assignment** if technician already has 8 pending tasks
   - Must complete or reassign existing tasks before assigning new ones
   - Exception: ADMIN can override with special confirmation (for emergencies)
2. **Workload Indicators**: System shows workload for reference
   - âœ“ Green (0-3 tasks): Normal workload - no warning
   - âš ï¸ Yellow (4-5 tasks): High workload - warning shown, assignment allowed
   - ğŸŸ  Orange (6-7 tasks): Very high workload - strong warning, confirmation required
   - ğŸ”´ Red (8 tasks): Maximum reached - **assignment blocked** (cannot assign more)
3. **Warning/Blocking Dialog**: 
   - When assigning to technician with 4-5 tasks â†’ Warning shown but assignment allowed
   - When assigning to technician with 6-7 tasks â†’ Strong warning with confirmation required
   - When technician has 8 tasks â†’ **Assignment blocked**, message: "This technician has reached maximum capacity (8 tasks). Please complete or reassign existing tasks first."
   - Admin override option (for emergencies only) - requires justification
4. **Admin Discretion**: Admin can decide to assign based on:
   - Urgency of task (URGENT tasks may justify high workload)
   - Priority level
   - Technician expertise/specialization
   - Current workload distribution
   - **Exception**: Even with admin discretion, cannot exceed 8 task limit (unless override)
5. **Notification**: Technicians receive notification when new task is assigned (even if they have existing tasks)

### Reassignment Rules:
- Can reassign task to different technician at any time (before completion)
- Original technician is notified of reassignment
- New technician sees task in "My Tasks"
- Status remains ASSIGNED, only `assigned_to` and `assigned_at` are updated
